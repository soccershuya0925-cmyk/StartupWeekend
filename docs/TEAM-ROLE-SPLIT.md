## 4. 他のエンジニア／メンバーへの依頼（詳細）

### 依頼① エンジニア①（HTMLプロト作者）｜方向性すり合わせ＋フロント
**ゴール**：A/B どちらで行くかの判断材料を出し、採用版のフロントを一緒に進める。
**やること**
1. HTML版の現状を共有（実装済み機能・未実装機能のリスト）
2. PR #1 の Next.js版をローカルで動かして比較（`npm install && npm run dev`）
3. **技術比較メモ**を1枚出す：両者の機能差分／HTML→Next移植 or Next→HTML移植のコスト感
**着手**：このリポを clone → `git fetch && git switch feat/nextjs-app` で Next.js版を確認
**完了条件**：比較メモが PR #1 にコメントされ、リーダー判断に使える状態

### 依頼② バックエンド／データ担当｜永続化＆AIの本番化
**ゴール**：デモを超えて「実際に使える」土台を用意。
**やること**
1. 現状 localStorage 保存 → 将来の DB 化（Supabase等）の**設計叩き台**を1枚（MVPは localStorage のままで可、設計だけ先に）
2. **OpenAI APIキーの取得・管理方針**を決める（誰のキー／環境変数 `OPENAI_API_KEY`／Vercel に登録）
3. **実レシートでの解析検証**：実際のレシート写真3枚で `/receipt` → `/api/receipt` が正しく食材を抽出するか確認、プロンプト調整の要否を報告
**着手**：`cp .env.local.example .env.local` してキーを入れ `npm run dev` → /receipt
**完了条件**：実レシート3枚で食材抽出が動く／キー管理方針が文書化

### 依頼③ デザイン／UX｜見た目とキャラクター
**ゴール**：発表で「おっ」となるビジュアル。
**やること**
1. キャラクター5段階（🥚→🍳→👨‍🍳→🏆→⭐）の**実イラスト or 統一感のある絵**
2. 配色・フォント・ロゴの方向性（現状は緑ベースの仮デザイン）
3. 主要4画面（ホーム/冷蔵庫/料理記録/キャラ）の**見た目フィードバック**
**完了条件**：キャラ画像 or デザイン指針が共有され、フロントに反映できる状態

### 依頼④ マーケ／検証｜ユーザーと発表
**ゴール**：「本当に欲しがられるか」の裏付けと、伝わる発表。
**やること**
1. 福岡の一人暮らし大学生に**インタビュー/アンケート**（食品ロスの実態・既存サービス不満・課金許容額）
2. スタンプ**提携先候補**の洗い出し（近隣スーパー/飲食店）
3. **発表スライド／デモ台本**（課題→ソリューション→デモ→市場→ビジネスモデル）
**完了条件**：検証結果サマリ＋発表ドラフトが揃う

## 5. 進め方のルール（全エンジニア共通）

- **main へ直接 push しない**。必ずブランチを切って PR（main にはチームのHTML版が乗っているため）
- 採用版の画面実装は **担当画面ごとに1ブランチ・1PR**
- PR は CI（`next build`）が緑になってからレビュー依頼
- 詰まったら PR #1 か Slack（or チームのチャット）で相談
